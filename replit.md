# AutoAdvertise Dashboard

A premium Discord automation SaaS. Users log in with Discord OAuth, redeem a license key, then manage multiple Discord accounts that automatically send advertising messages in channels.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API + Discord bot server (port 8080)
- `pnpm --filter @workspace/dashboard run dev` — run the React frontend (port 23183)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + Tailwind CSS + Framer Motion + Wouter
- Backend: Express 5 + Discord.js v14
- Database: MongoDB Atlas (Mongoose ODM)
- Auth: Discord OAuth2 + express-session (stored in MongoDB via connect-mongo)
- API codegen: Orval (from OpenAPI spec → React Query hooks + Zod schemas)
- Bot: Discord slash commands (/genkey, /revokekey, /keyinfo, /listkeys, /extendkey, /resetkey, /deletekey)
- Build: esbuild (CJS bundle for server)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for all API contracts)
- `lib/api-client-react/src/generated/` — Generated React Query hooks (don't edit manually)
- `lib/api-zod/src/generated/` — Generated Zod validation schemas (don't edit manually)
- `artifacts/api-server/src/` — Express server + Discord bot
  - `src/models/` — Mongoose models (User, License, Account, Log, Settings)
  - `src/routes/` — Express route handlers
  - `src/bot/` — Discord bot + slash command files
  - `src/lib/` — Utilities (MongoDB, auth middleware, advertiser engine, key generator)
- `artifacts/dashboard/src/` — React frontend (pages, components)

## Architecture Decisions

- MongoDB Atlas used instead of Replit's built-in PostgreSQL (user's preference)
- Discord bot runs inside the same process as the Express server for simplicity
- Auto-advertising engine uses setInterval per account, stored in memory; active accounts are resumed on server restart
- Sessions stored in MongoDB via connect-mongo (not Postgres)
- License keys are format XXXX-XXXX-XXXX-XXXX (uppercase alphanumeric)

## Product

- Landing page with Discord OAuth login
- License key redemption (keys generated by `/genkey` Discord slash command)
- Dashboard: stats overview, recent activity
- Accounts: add/edit/delete Discord accounts with token, channel IDs, messages, cooldown
- Start/Stop auto-advertising per account
- Logs: filterable activity log
- Settings: global config
- License info: plan, expiry, status

## Gotchas

- SESSION_SECRET is a Replit Secret (already configured)
- MongoDB URI, Discord bot token, and Discord client secret are stored as env vars (shared environment)
- DISCORD_CLIENT_ID and ADMIN_DISCORD_IDS are stored as env vars
- After any OpenAPI spec change, always run codegen before editing routes
- esbuild bundles everything — check build.mjs externals if new native modules are added
- Discord OAuth redirect URI must be added to the Discord application: `https://<your-domain>/api/auth/discord/callback`
- Admin Discord user IDs for bot commands: set via ADMIN_DISCORD_IDS env var (comma-separated)
